Introduction to SQL

SQL

SQL Parts

  • DML（Data Manipulation Language）：用于查询/检索及对元组的增删改查（SELECT、INSERT、DELETE、UPDATE）
  • DDL（Data Definition Language）：用于定义/删除/修改模式、视图和索引等结构
  • 完整性（Integrity）：指定完整性约束的命令
  • 视图定义（View definition）：用于定义视图的命令
  • 事务控制（Transaction control）：用于指定事务开始与结束的命令
  • 嵌入式与动态 SQL（Embedded SQL and dynamic SQL）：在编程语言中嵌入的 SQL 语句
  • 授权（Authorization）：与权限管理相关的命令

下表是常见的功能说明及其命令

功能类别 说明 常见命令
数据查询 (data query) 用于检索/查询数据 SELECT
数据操作 (data manipulation, DML) 对元组进行增删改查 INSERT, DELETE, UPDATE
数据定义 (data definition, DDL) 定义/修改/删除模式、视图、索引等结构 CREATE, DROP, ALTER（on schema）
数据控制 (data control) 权限与授权管理 GRANT, REVOKE
事务处理 (transaction processing) 指定事务的开始与结束、保证一致性 BEGIN TRANSACTION, COMMIT, ROLLBACK
指针/游标控制语言 (CCL) 游标声明与逐行处理当前记录 DECLARE CURSOR, FETCH INTO, UPDATE … WHERE CURRENT

SQL Data Definition

DDL（数据定义语言）用于描述关系（relation）的元信息与约束。

  • 包含内容：
    • 每个关系的模式（schema）
    • 每个属性对应的值类型
    • 完整性约束（例如 age > 0）
    • 为关系维护的索引
    • 关系的安全性与授权信息
    • 关系在磁盘上的物理存储结构

Basic Schema Definition: Create Table

  • 主题：使用 CREATE TABLE 定义关系模式与完整性约束（DDL 的一部分）。

  • CREATE TABLE 的一般形式：

    • create table r (A1 D1, A2 D2, …, (integrity-constraint1), … )
    • 其中 r 为关系名，Ai 为属性名，Di 为属性的数据类型。

  • DDL 在表定义中还描述：模式、属性类型、完整性约束、索引、授权与物理存储等元信息。

  • 完整性约束类型：

    • 主键（primary key）：保证行唯一性。
    • 外键（foreign key … references …）：维护引用完整性。
    • NOT NULL：禁止空值。
    • 这些约束阻止任何违反完整性的插入/更新/删除操作。

  • 流程要点：先对参与的关系做笛卡尔积（或连接），再按谓词 P 选择满足的元组，最后对所需属性做投影。

The Select Clause

SELECT 子句（对应关系代数的投影）

  • 作用：列出要返回的属性（列），即对关系做投影，形式如 select A1, A2, ... from ...

  • 等价关系代数表示：

    \[ \Pi_{A1,A2,\dots,An}(\sigma_{P}(r_1\times r_2\times\dots)) \]

  • 示例：查找所有教师的姓名 select name from instructor

  • 表格要素：attributes（列）和 tuples/rows（行）

  • 注意：名字（列名）大小写不敏感（Name ≡ NAME ≡ name）

  • 无 FROM 的字面量：select '437' 返回一个单列单行的表；可用别名命名列（select '437' as FOO）。

  • 带 FROM 的字面量：select 'A' from instructor 返回一列 N 行的表，每行值为 'A'（N 为表中元组数）。

SQL 查询结果可以控制重复元素

  • 要点：
    • SQL 查询结果允许重复元组（duplicates）。
    • 使用 DISTINCT 强制去除重复。
    • 使用 ALL 明确保留重复（默认行为）。
  • 示例：
    • select distinct dept_name from instructor;
    • select all dept_name from instructor;

星号 (*)：在 SELECT 子句中表示返回所有属性（例如 select * from instructor）。

注意：实际应用中尽量避免在 SELECT 中使用 *，以防影响索引与查询性能。

SELECT 子句可以包含算术表达式（+, -, *, /），对常量或元组的属性进行运算。

  • 示例：select ID, name, salary/12 from instructor 返回与 instructor 相同的结果集，但 salary 值被除以 12。
  • 可用 AS 为表达式重命名，例如 select ID, name, salary/12 as monthly_salary，将计算结果列命名为 monthly_salary。

Where Clause Predicates

  • Between 比较运算符：用于范围查询，含义为大于等于下限且小于等于上限。

    • 例：查找薪水在 \(90{,}000\) 到 \(100{,}000\)（含端点）的教师

  • 元组比较（Tuple comparison）：可以将多个属性作为一个元组进行比较，用于跨表或常量的多列匹配。

    • 例：匹配 instructor 的 (ID, dept_name) 与 teaches 的 (ID, ‘Biology’)

The From Clause

FROM 子句列出参与的关系，对应关系代数中的笛卡尔积（Cartesian product）。

  • 作用：例如 select * from instructor, teaches 会生成每个 instructor–teaches 的所有组合，结果包含两表的所有属性。
  • 属性重名：若存在相同属性名（如 ID），结果中的属性需用关系名前缀区分（例如 instructor.ID）。
  • 结论：笛卡尔积本身直接用处有限，通常需配合 WHERE 子句的连接或筛选条件才能得到有意义的结果。

The Rename Operation

重命名操作（Rename operation）——使用 as 子句为关系或属性起别名：old-name as new-name。

  • 目的与作用：

    • 引入元组变量（关系别名），便于自连接（self-join）和区分同名属性；
    • as 关键字可选（例如 instructor as T 或 instructor T 均可）；
    • 有助于在 WHERE 中引用不同实例的同一表（如 T.salary 与 S.salary）。

String Operations

  • 字符串匹配操作（LIKE）的用法与特殊通配符：
    • %：匹配任意长度的子串（任意字符串）。
    • _：匹配任意单个字符。
  • 示例：查找名字包含子串 “dar” 的教师：
    • select name from instructor where name like '%dar%'
  • 若要匹配字面量 ‘%’，需使用转义字符（示例用反斜杠）：
    • like '100\%' escape '\'
  • 模式匹配区分大小写。
  • 常见匹配示例：
    • 'Intro%'：以 “Intro” 开头的任意长度字符串；
    • '%Comp%'：包含子串 “Comp” 的任意字符串；
    • '___'：恰好 3 个字符的字符串（下划线代表任意单字符）；
    • '___%'：至少 3 个字符的字符串。
  • 常用字符串操作包括：
    • 连接（使用 ||）；
    • 大小写转换（upper/lower）；
    • 计算长度；
    • 提取子串等。

Ordering the Display of Tuples

对元组显示结果进行排序（Ordering the Display of Tuples）

  • 主要点：
    • 使用 ORDER BY 对查询结果排序，例如：select distinct name from instructor order by name。
    • 指定排序方向：ASC（默认，升序）或 DESC（降序），例如 order by name desc。
    • 可按多个属性排序，按先后顺序决定主次，例如：order by dept_name, name 或 order by dept_name asc, name desc。
    • 注意：对大数据文件使用数据库的 ORDER BY 可能开销很大（排序代价高）。

Null Values

  • Null 表示“未知值”或“值不存在”。
  • 任何包含 null 的算术表达式结果为 null。
  • 使用谓词 is null 检查空值。

    • 示例：查找 salary 为 null 的教师：

SQL 中任何与 NULL 参与的比较都被视为未知（unknown），例如 5 < NULL、NULL <> NULL、NULL = NULL 都为 unknown。

  • 布尔运算扩展规则（处理 unknown）：
    • AND：true AND unknown = unknown，false AND unknown = false，unknown AND unknown = unknown
    • OR：unknown OR true = true，unknown OR false = unknown，unknown OR unknown = unknown
  • WHERE 子句：当谓词计算结果为 unknown 时，视为 false（即该行不会被选中）。
  • 实践建议：尽量避免在属性中引入 NULL（以免影响索引等），可用默认值替代。
