xv6 除零异常处理流程分析

当用户态程序发生除零异常时，CPU 会完成一系列硬件操作，包括保存寄存器、切换到内核栈，并最终跳转到在 vectors.S 中预设好的中断服务例程（ISR）vector0。vector0 会压入一个假的错误码（因为除零异常硬件不产生错误码）和中断号 T_DIVIDE，然后跳转到 trapasm.S 文件中的 alltraps 函数，后续的软件处理流程正式开始。

1. 关键数据结构

在异常处理流程中，主要涉及以下几个关键的结构体：

  1. struct trapframe (定义于 x86.h):
    • 作用: trapframe 是理解中断处理流程的核心。它定义了在进入内核时，需要保存在内核栈上的寄存器集合。当从用户态陷入内核态时，硬件和 alltraps 中的代码会共同在当前进程的内核栈上构建一个 trapframe 结构体。这个结构体完整地记录了中断发生时用户程序的所有上下文信息，包括通用寄存器、段寄存器、程序计数器（EIP）、栈指针（ESP）等。后续的 trap 函数会直接以指向这个结构体的指针作为参数，从而访问和修改被中断进程的状态。当需要返回用户态时，内核会利用这个结构体中的信息恢复现场。
  2. struct proc (定义于 proc.h):
    • 作用: 这是 xv6 的进程控制块（Process Control Block, PCB）。它包含了管理一个进程所需的所有信息。在异常处理中，proc 结构体的以下字段尤为重要：
      • tf: 指向当前进程的 trapframe。当进程从用户态切换到内核态时，tf 会指向内核栈上保存的 trapframe。
      • state: 表示进程的当前状态（如 RUNNING, SLEEPING, ZOMBIE 等）。在除零异常处理中，这个状态会从 RUNNING 变为 ZOMBIE。
      • killed: 一个标志位。当该位置为 1 时，表示该进程需要被终止。在除零异常处理中，内核会设置这个标志位来标记该进程为“死亡”状态。

2. 关键函数与执行流程

从 alltraps 开始，执行流程如下：

alltraps -> trap -> exit

  1. alltraps 函数 (位于 trapasm.S):
    • 作用: 这是一个汇编例程，是所有 Trap（中断、异常、系统调用）的统一入口。
    • 执行内容:
      1. 保存寄存器: alltraps 首先将 ds, es, fs, gs 等段寄存器和 eax, ecx, edx, ebx, esi, edi, ebp 等通用寄存器依次压入内核栈。
      2. 构建 trapframe: 结合硬件已经压栈的 ss, esp, eflags, cs, eip 以及 vector0 压入的 trapno 和 err，这些压栈操作共同在内核栈的顶部形成了一个完整的 trapframe 结构体。
      3. 切换到内核数据段: 将 ds 和 es 设置为内核数据段选择子 (SEG_KDATA<<3)。
      4. 调用 trap 函数: 将当前栈顶指针（即 trapframe 的地址）作为参数压栈，然后调用 C 语言实现的 trap 函数。
  2. trap 函数 (位于 trap.c):
    • 作用: 这是内核处理所有 Trap 的核心 C 函数。它通过检查 trapframe 中的中断号 tf->trapno 来分发处理不同的事件。
    • 执行内容:
      1. 获取进程信息: trap 函数首先会确认 trapframe 是否为空，并获取当前 CPU 正在运行的进程（myproc()）。
      2. 保存 trapframe 地址: 将 trapframe 的地址保存到当前进程的 proc 结构体中 (myproc()->tf = tf;)。
      3. 分发处理: 使用 switch 语句根据 tf->trapno 的值进行判断。
      4. 处理除零异常: 对于除零异常，tf->trapno 的值是 T_DIVIDE。在 switch 的 case T_DIVIDE: 分支中，内核认为这是一个不可恢复的错误。
        • 它会打印一条错误信息，指明是哪个进程（PID）在哪个地址（EIP）发生了除零异常。
        • 然后将当前进程的 killed 标志位设置为 1 (myproc()->killed = 1;)。
      5. 检查 killed 标志: 在 trap 函数的末尾（switch 语句之后），会检查当前进程的 killed 标志位。由于上一步已经将其设置为 1，该检查会通过。
      6. 调用 exit: 如果 killed 标志位为 1，trap 函数会调用 exit() 函数来终止当前进程。之后，trap 函数不会返回。
  3. exit 函数 (位于 proc.c):
    • 作用: 负责终止当前进程，并进行资源回收和状态转换。
    • 执行内容:
      1. 关闭文件描述符: 遍历并关闭该进程打开的所有文件。
      2. 回收内存: 释放进程的用户空间内存。
      3. 处理子进程: 将该进程的所有子进程过继给 init 进程。
      4. 唤醒父进程: 如果父进程正在 wait() 等待该子进程退出，则唤醒父进程。
      5. 改变进程状态: 将当前进程的 state 修改为 ZOMBIE。处于 ZOMBIE 状态的进程保留了 proc 结构体，以便父进程可以读取其退出状态，但其占有的其他资源（如内存、文件）已被释放。
      6. 调度新进程: 调用 sched() 函数，放弃 CPU，让调度器 scheduler() 选择另一个进程来运行。从此，发生除零异常的进程彻底停止执行。

3. 用户程序的状态变化

发生除零异常的用户程序，其 proc->state 会经历以下变化：

  1. RUNNING -> ZOMBIE:
    • 在发生异常之前，该进程处于 RUNNING 状态（或 RUNNABLE，在等待被调度）。
    • 当异常发生，内核接管控制权，在 trap 函数中将其 killed 标志位置 1。
    • 随后调用的 exit 函数会将进程状态最终设置为 ZOMBIE。
    • 这个 ZOMBIE 进程会一直存在，直到其父进程调用 wait() 来回收它（读取其退出状态并释放 proc 结构体）。

总结

总而言之，当 xv6 的用户程序发生除零异常时，系统会认为这是一个致命错误，处理流程如下：硬件和汇编代码首先保存现场并构建 trapframe，然后调用 trap 函数。trap 函数识别出是除零异常后，将进程标记为 killed，并立即调用 exit 函数。exit 函数负责清理该进程占用的资源，并将其状态设置为 ZOMBIE，最后调用调度器交出 CPU。整个过程中，发生异常的进程被强制终止，无法恢复执行。
